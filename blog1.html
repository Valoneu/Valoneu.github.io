My Journey vibecoding this site

Posted by Valoneu on April 6th, 2025

This portfolio site you're looking at came together through a process recently dubbed "vibecoding". It wasn't about meticulously hand-crafting every line of code in the traditional sense. Instead, it was an exploration into AI-assisted development. This post details how I vibecoded this site using an AI code assistant, and what I learned along the way.

So, what exactly is vibecoding? As defined recently, it's an AI-dependent technique. You describe a problem or desired feature in natural language to a large language model (LLM) tuned for coding. The AI then generates the code. My role shifted significantly – less manual coding, more guiding the AI, prompting effectively, testing the output, and iteratively refining the generated source code until it matched the 'vibe' I was going for.

Prompting the Visual Flow with CSS

The process started with high-level prompts. I described the overall structure: a sticky navigation bar, sections for Home, About, Skills, Projects, Blog, and a footer. I specified the desire for a dark theme as the default, using Roboto Mono and Space Mono fonts. The AI generated the initial HTML structure and the CSS including custom properties for theming.

Getting the layouts right, like the project cards and skills grid, involved back-and-forth prompting. I'd ask for a responsive grid layout, then refine it by asking the AI to adjust gaps, alignment, or behavior on different screen sizes. The animated intro section with the morphing shapes was a prime example: I described the effect ("floating, abstract, subtly animated background blobs using theme colors") and iterated with the AI, asking it to tweak animation timings, shapes (`border-radius` percentages), and blur effects until it felt right.

Generating JavaScript for Interaction

The site's interactivity was also largely AI-generated. For the logo's text scramble effect, I prompted something like: "Create a JavaScript function that scrambles the text of an H1 element on mouseover, revealing the original text letter by letter, and reverts it on mouseout." The AI provided the initial function, which I then tested and potentially asked for modifications (like adjusting the speed).

Similarly, the theme toggle involved asking the AI to write JavaScript that detects the button click, toggles a `data-theme` attribute on the `` tag, and uses `localStorage` to remember the user's preference. The project filtering logic was requested by describing the need to filter elements based on `data-category` attributes when corresponding filter buttons are clicked. The AI generated the event listeners and filtering functions, which I then integrated and tested.

Learnings from the AI Vibes

Vibecoding with an AI assistant is a fascinating and powerful way to build, especially for personal projects where rapid iteration is key. It significantly speeds up development but introduces its own set of challenges and required skills.

Key takeaways from this process:

  • Prompt Engineering is Crucial: The quality of the AI's output is highly dependent on the clarity and specificity of the prompts. Learning how to effectively describe requirements to the AI is a skill in itself.
  • Refinement is Non-Negotiable: AI-generated code is rarely perfect on the first try. Expect to spend time testing, debugging, and guiding the AI through multiple revisions to get the desired functionality and style.
  • Understanding Code Still Matters: While the AI writes the code, you still need to understand it to effectively debug, integrate, and refine it. You can't blindly trust the output.
  • AI Excels at Boilerplate and Known Patterns: It's fantastic for generating standard structures, functions (like theme toggles or event listeners), and CSS layouts quickly.
  • Beware of Subtle Bugs: AI can sometimes introduce subtle logical errors or edge cases that require careful testing to uncover.

Overall, vibecoding this site was an insightful experience into the evolving landscape of software development. It allowed me to translate my vision into a functional website much faster than traditional methods might have allowed, while simultaneously honing my skills in guiding and collaborating with an AI code assistant. It's a different way of working, but one with immense potential.
